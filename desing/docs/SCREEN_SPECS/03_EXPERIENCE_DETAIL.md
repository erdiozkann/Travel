# SCREEN SPEC: Experience Detail
Screen ID: 03
Route: `/explore/experience/:experienceId` or `/map/experience/:experienceId`
Design Reference: `01_customer_core/experience_details_-_light`

---

## 1. Purpose

The Experience Detail screen is the **conversion page** for bookable activities. It provides:
- Full experience information (description, duration, price, availability)
- AI-generated summary with confidence label
- Trust signals (reviews, ratings, host/provider info)
- Clear pricing visibility
- Primary CTA: **"Book experience"** → Stripe Checkout

This screen answers: **"Should I book this experience?"**

## 4. State Handling

### 4.1 Local State (UI)
| State | Type | Default |
|---|---|---|
| `experienceId` | uuid | from route |
| `experience` | Experience? | null |
| `aiSummary` | AISummary? | null |
| `reviews` | List<Review> | [] |
| `isLoading` | bool | true |
| `isFavorited` | bool | false |
| `isBooking` | bool | false |
| `mediaIndex` | int | 0 |
| `isDescriptionExpanded` | bool | false |

### 4.2 Cached State
| State | Source | TTL |
|---|---|---|
| `experience` | DB + cache | 1 hour |
| `aiSummary` | ExperienceSummaryAgent | 7 days (cache_ttl: 604800) |
| `reviews` | DB | 15 minutes |

### 4.3 State Transitions
```
[Screen Load]
  → fetch experience from cache or DB
  → fetch AI summary (cached)
  → fetch reviews (first page)
  → isLoading = false

[Book CTA Tap]
  → if not authenticated → redirect to /auth/login?redirect=current
  → isBooking = true
  → call Edge Function: create_experience_checkout
  → redirect to Stripe Checkout URL
  → handle return (success/cancel)

[Favorite Toggle]
  → if not authenticated → prompt login
  → toggle isFavorited
  → update DB (saved_items table)

[Image Tap]
  → open fullscreen gallery modal
```

## 8. Stripe Checkout Flow

### 8.1 Flow Diagram
```
[User taps "Book experience"]
    ↓
[Auth check]
    ├── Not logged in → /auth/login?redirect=current
    └── Logged in → continue
    ↓
[Call Edge Function: create_experience_checkout]
    Request: { experience_id, quantity, user_id }
    Response: { checkout_url, session_id }
    ↓
[Redirect to Stripe Checkout]
    ↓
[Stripe handles payment]
    ├── Success → Stripe webhook → create booking_experience (status: paid)
    └── Cancel → return to app, no record created
    ↓
[Return to app]
    ├── Success URL: /booking/success?session_id=xxx
    └── Cancel URL: /explore/experience/:id (current page)
```

### 8.2 Edge Function: create_experience_checkout
**Purpose:** Create Stripe Checkout Session, return URL

**Input:**
```json
{
  "experience_id": "uuid",
  "quantity": 1,
  "selected_date": "2026-03-15",
  "selected_time": "10:00"
}
```

**Output:**
```json
{
  "checkout_url": "https://checkout.stripe.com/...",
  "session_id": "cs_xxx"
}
```

**Server-side actions:**
1. Validate experience exists and is available
2. Calculate price
3. Create Stripe Checkout Session
4. Return URL (do NOT create booking yet)

### 8.3 Webhook: stripe_webhook
**Purpose:** Confirm payment, create booking record

**On `checkout.session.completed`:**
1. Extract `experience_id`, `user_id`, `amount_paid`
2. Create `booking_experience` record with `status: paid`
3. Send confirmation (push notification / email)

## 10. Edge Cases

### 10.1 Experience Not Found
| Condition | Behavior |
|---|---|
| Invalid ID | Show error: "Experience not found" + back button |
| Deleted/suspended | Show error: "This experience is no longer available" |

### 10.2 Loading States
| Condition | Behavior |
|---|---|
| Initial load | Skeleton UI (image placeholder, text lines) |
| AI summary loading | "Generating summary..." with spinner |
| Reviews loading | Skeleton review cards |
| Checkout loading | Full-screen loading overlay |

### 10.3 Offline
| Condition | Behavior |
|---|---|
| Cached experience | Show cached data + "Offline" banner |
| No cache | Show error: "No connection" |
| Booking attempt offline | Block + show "Connection required to book" |

### 10.4 Authentication
| Condition | Behavior |
|---|---|
| Not logged in + tap Book | Redirect to login with return URL |
| Not logged in + tap Favorite | Show login prompt (modal) |
| Session expired during checkout | Stripe handles, return to login if needed |

### 10.5 Payment Failures
| Condition | Behavior |
|---|---|
| User cancels at Stripe | Return to detail page, no action |
| Payment declined | Stripe shows error, user can retry |
| Webhook fails | Retry mechanism server-side, user sees "Processing..." |

### 10.6 Availability
| Condition | Behavior |
|---|---|
| No available dates | Show "Currently unavailable" + disable CTA |
| Sold out for selected date | Show "Sold out" + suggest alternatives |

### 10.7 Sponsored Content
| Condition | Behavior |
|---|---|
| Experience is sponsored | Show "Sponsored" badge in header |
| Sponsored in related | Show badge on related cards |

## 14. Design Decisions (APPROVED)

Inherited from previous specs:
1. **Full-screen city selector**: Used for location context
2. **Price always as range**: Never exact (GUARDRAILS.md)

Experience-specific:
1. **AI Summary always visible**: Not behind toggle
2. **Confidence badge**: Always shown with AI content
3. **Reviews section**: Top 3-5 visible, "See all" for more
4. **Sticky CTA bar**: Always visible at bottom
5. **Stripe Checkout redirect**: Not in-app payment form
